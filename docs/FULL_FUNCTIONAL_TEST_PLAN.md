# 全功能测试规划

本文档定义“全功能测试”的范围、数据准备、执行顺序和通过标准。它不是简单页面冒烟；完整通过需要测试库、测试上传目录、至少 4 类账号或等价的本地 storage state。

## 测试分层

### L0 静态与构建

目标：确认代码、Prisma schema、迁移和构建链路可复现。

必跑命令：

```bash
npm run check
npx prisma migrate diff --from-migrations prisma/migrations --to-schema prisma/schema.prisma --exit-code
npm run build
```

通过标准：

- Prisma validate、generate、TypeScript、scripts TypeScript、ESLint、`git diff --check` 全部通过。
- migration diff 无差异。
- build 成功；warning 必须记录并分级。

### L1 页面与权限冒烟

目标：确认主要面板在桌面和移动端可进入、不 500、不出现 Next error overlay、不横向溢出。

入口：

- `/`
- `/profile`
- `/feedback`
- `/procurement`
- `/procurement/list`
- `/procurement/dashboard`
- `/procurement/new`
- `/procurement/workshop-fee`
- `/progress`
- `/progress/list`
- `/progress/dashboard`
- `/progress/archive`
- `/progress/new`
- `/admin`
- `/admin/roles`
- `/admin/acceptance`
- `/admin/budget-pools`
- `/admin/project-templates`
- `/admin/reminders`
- `/admin/system`
- `/not-exists-for-playwright`
- `/uploads/playwright-missing-file.png`

通过标准：

- 未登录访问受保护页面不 500，可跳登录或显示无权限。
- 登录后上述页面不跳登录。
- `/uploads/...` 未登录不可直接读取，首跳为 3xx/401/404。
- 桌面 `1440x1000` 和移动端 Pixel 5 无横向溢出。

执行：

```bash
PLAYWRIGHT_BASE_URL=http://127.0.0.1:3002 \
PLAYWRIGHT_STORAGE_STATE=/path/to/playwright-liqixuan-storage.json \
PLAYWRIGHT_ADMIN_STORAGE_STATE=/path/to/playwright-admin-storage.json \
npm run test:e2e
```

### L2 单账号浅交互

目标：确认常用筛选、URL 状态和基础点击没有明显回退。

场景：

- 反馈中心：点击“全部”，点击反馈项，URL `selected` 与详情保持稳定。
- 项目列表：点击“已超期”，点击“只看自己”，URL `deadline` 与 `mine` 保留。
- 任务看板：切换风险筛选、紧急度筛选、全队/只看自己。
- 采购列表：切换状态筛选，进入草稿/订单详情。
- 管理面板：进入角色、验收条例、提醒规则、系统同步页面。

通过标准：

- URL query 可刷新复现。
- 列表或空状态文案正常。
- 控制台无 uncaught error。

### L3 业务闭环

完整通过必须准备可写测试数据，并在独立 PostgreSQL 测试库执行。不得使用生产 3000 和生产数据库。

#### 采购

账号：

- 申请人
- 车组组长
- 技术组组长
- 老师或超级管理员
- 报销员

场景：

1. 新建采购申请，必填校验显示中文错误。
2. 保存草稿，从列表进入编辑，再保存草稿。
3. 草稿编辑后提交申请。
4. 管理审核：车组组长通过、技术组组长通过。
5. 老师审核通过。
6. 申请人上传发票、每行实物照片，生成验收清单。
7. 缺少签名、发票、照片时失败并清理临时上传。
8. 报销员上传截图。
9. 申请人确认报销，订单完成。
10. 驳回路径：管理驳回、老师驳回、报销退回、申请人重传。
11. 工坊加工费：创建、草稿、提交、审批。
12. 附件权限：匿名 401/3xx，越权 403/404，有权可读，伪 MIME/超大小拒绝。

重点断言：

- 每次状态变化写入动态或审批记录。
- outbox 有对应通知，重复 drain 不重复发送成功项。
- 失败路径不留下孤儿 `FileAsset` 或磁盘文件。

#### 进度项目

账号：

- 项目负责人
- 项目参与人
- 任务负责人
- 项管
- 超级管理员
- 对应车组/技术组组长

场景：

1. 创建项目，选择负责人、参与人员、阶段模板。
2. 编辑项目参与人员，通知覆盖新旧参与人和管理角色。
3. 启动项目，当前阶段 DDL 在详情和列表展示。
4. 阶段提交验收，审批通过/驳回，阶段 DDL 历史记录可见。
5. 当前阶段延期申请：项目负责人申请，项管/超管审批，后续阶段顺延。
6. 单独修改阶段 DDL：申请、项目负责人审批，历史记录可见。
7. 完成项目：阶段和任务未完成时按钮禁用并显示原因；满足条件后完成。
8. 取消项目：未结束任务转为 `PROJECT_CANCELED`，完成/归档任务保持原状态。

重点断言：

- 所有已登录用户可查看项目。
- 只有管理员/项目相关人员能操作。
- 只看自己包含项目负责人、参与人、阶段负责人、任务负责人、申请人。

#### 任务

场景：

1. 管理员直接创建任务，任务技术组多选，`宣运` 在 `通用` 上方。
2. 参与人员发起任务创建申请，管理员通过后真实任务生成。
3. 任务编辑不能直接修改 DDL，只能申请修改 DDL。
4. 任务负责人或项目负责人申请修改 DDL，审批通过/驳回。
5. 任务状态：待办、开始、提交交付、验收通过、验收驳回、归档、重启。
6. 周报：有权限账号提交；详情显示本周周报剩余/今日/逾期/已提交。
7. 风险同步：新增风险、展示未解决风险、解除风险、历史保留。
8. 删除申请：参与人/负责人申请，管理员通过软删除或驳回。

重点断言：

- 任务通知收件人包含项目车组/技术组组长、任务技术组组长、项目参与人、任务负责人、项管、项目负责人。
- `PROJECT_CANCELED` 是终止态，不能继续提交、验收、编辑、重启。
- 看板逾期/今日/临期/风险筛选正确。

#### 反馈

场景：

1. 新建反馈，图片类型、数量、单张 20MB、合计 50MB 校验。
2. 回复反馈，管理员/提交人双方可见。
3. 状态切换：开放、处理中、已关闭。
4. 点击“全部”后选择已关闭/活动反馈，筛选不被 URL selected 强制覆盖。
5. 直接打开 `/feedback?selected=<closedId>` 可进入能看到该反馈的筛选视图。

#### 管理员

场景：

1. 角色增删改，重新登录后权限生效。
2. 验收条例模板增删改，任务表单可引用。
3. 项目模板增删改，项目创建可引用。
4. 预算池增删改，采购触发预算提醒。
5. 进度提醒规则修改，手动扫描生成 outbox；重复扫描不重复入队。
6. 系统同步：飞书用户同步失败/成功均有明确提示。

### L4 并发与一致性

场景：

- 订单号并发创建。
- 管理审核双击/双账号同时通过。
- 任务 DDL 双审批。
- 任务验收双击。
- outbox drain 部分失败后只重试失败收件人。
- cron/app 多实例同时扫描提醒。

通过标准：

- 使用 `updateMany` 条件锁或唯一键保证只推进一次。
- 重复请求返回中文可读错误，不产生重复状态记录。

## 测试数据建议

至少准备：

- 5 个测试用户：申请人、车组组长、技术组组长、项管/超管、报销员。
- 2 个车组、3 个技术组，含 `宣运`。
- 采购订单：草稿、待管理审核、待老师审核、待上传凭证、待报销截图、待确认、已完成、已驳回。
- 项目：未启动、进行中、待验收、已完成、已取消。
- 阶段 DDL：已超期、今日到期、临期、正常、无 DDL。
- 任务：待办、进行中、待验收、已完成、已归档、项目已取消、带风险、带 DDL 申请。
- 反馈：开放、处理中、已关闭，含图片附件。
