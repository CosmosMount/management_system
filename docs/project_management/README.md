# 项目管理核心业务逻辑文档

本目录描述项目管理模块的核心业务逻辑，用于指导产品沟通、流程校准、权限配置和后续扩展。这里不是技术实现文档；框架、数据库、部署、环境变量和代码结构见 [`../TECH.md`](../TECH.md)。

## 文档范围

本目录只覆盖项目管理模块，包括项目创建、项目生命周期、阶段验收、任务执行、周报、交付验收、风险同步、归档和提醒。

系统内其他模块不在本目录展开。若其他模块与项目管理共用账号、角色或通知基础能力，本目录只说明这些能力在项目管理中的业务含义。

## 公共术语

- 项目：进度管理的主对象，承载阶段、任务、状态和最终结果。
- 阶段：项目内按顺序推进和验收的小生命周期节点。
- 任务：项目内可分配给一个或多个负责人的执行单元。
- 范围：权限生效边界，当前主要是车组、技术组和全局。
- 处理人：当前状态下有权推动业务对象进入下一环节的人。
- 证据：飞书文档链接、视频链接、周报、验收记录等可供审核或留档的材料。
- 通知不回滚业务：业务状态先完成保存，通知失败只记录问题，不撤销已经完成的业务动作。

## 维护原则

- 本目录只写业务逻辑，不展开代码结构、接口路径、数据库字段和部署方案。
- 若实现发生变化，先更新对应业务规则，再同步技术文档。
- 不把未来设想写成当前能力；未实现能力应写入扩展边界或待补规则。
