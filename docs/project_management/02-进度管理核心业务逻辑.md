# 进度管理核心业务逻辑

## 业务目标

进度管理分为项目管理和任务管理。项目管理关注项目生命周期和阶段验收；任务管理关注具体执行单元、交付、周报、风险和逾期。

## 项目生命周期

项目大生命周期如下：

```text
未开始 -> 进行中 -> 已完成
未开始/进行中 -> 已取消
```

| 状态 | 业务含义 |
|---|---|
| 未开始 | 项目已创建，尚未启动第一个阶段 |
| 进行中 | 项目已启动，阶段和任务可推进 |
| 已完成 | 全部阶段完成后，由项目负责人确认结束 |
| 已取消 | 项目非正常终止，不能继续推进 |

项目负责人是项目生命周期的常规修改人。项目进入“已完成”前，必须完成全部小阶段。

## 项目阶段

项目阶段是项目的小生命周期。阶段可以从模板生成，也可以在创建项目时自定义。

实车项目默认模板包含：

```text
研讨 -> 机电连线图 -> 机械图纸绘制 -> 发加工 -> 装车布线
-> 第一次上电验收 -> 基础功能验收 -> 功能实现 -> 留档
```

每个阶段包含：

- 阶段名称。
- 需要完成目标的文字描述。
- DDL。
- 一名或多名阶段负责人。
- 当前状态。
- 文档或文件归档链接。
- 提交、审批和操作留档。

项目启动后，第一个阶段进入进行中。阶段审批通过后，该阶段完成，并自动开启下一个未开始阶段。

## 阶段提交与审批

任一阶段负责人、项目负责人或项目参与人员均可提交阶段材料。材料可以是飞书文档链接，也可以是文件归档链接。

阶段审批规则：

- 项目负责人审批阶段提交。
- 默认禁止提交人审批自己的阶段提交。
- 如果提交人和项目负责人相同，且项目关闭自审，则由项管、超级管理员、匹配车组组长或匹配技术组组长审批。
- 审批通过后阶段完成；审批驳回后阶段回到进行中。
- 每次提交、审批和状态变化都必须留档。

项目可在创建时开启“允许项目负责人自审”。默认关闭。

## 任务

任务是项目管理的基本执行单元。任务可以归属于某个项目阶段，也可以不归属任何阶段。任务完成不受所属阶段是否开启影响。

任务包含：

- 任务目标和说明。
- 任务分类，例如测试、装车、研发、调试、审图、迭代。
- 紧急程度和重要程度。
- 一个或多个负责人。
- 车组和技术组。
- 期望达到或要求达到的定量/定性指标。
- 最晚完成时间。
- 是否需要线下确认。
- 是否需要定期周报。

## 任务状态

任务主流程如下：

```text
待办 -> 进行中 -> 待验收 -> 已完成 -> 已归档
```

任务可以有一个或多个负责人。任一任务负责人都可以开始任务、提交周报、提交交付和同步风险。任务完成不能直接手动标记，必须提交交付并经过验收。

## 任务交付与验收

任务负责人提交交付时，需要提供：

- 交付文档或归档链接。
- 关键数据链接，例如控制曲线、自瞄视频、装配照片或其他关键材料。
- 未达标原因，如有。
- 补充说明，如有。

验收人需要人工核对交付材料。审批通过后任务进入已完成；驳回后任务回到进行中。

## 周报与长周期任务

任一任务负责人可以提交周报，内容包括本周完成情况、风险同步、下周计划和可选文档链接。

长期任务通过“需要定期周报”字段标记。创建任务时可以手动勾选；截止时间较远的任务默认需要周报。

## 风险同步

任一任务负责人可以随时同步风险。风险同步会通知：

- 全部任务负责人。
- 项目负责人。
- 匹配车组组长。
- 匹配技术组组长。
- 项管。

风险同步不自动改变任务状态，但会在任务和看板中暴露风险标记。

## 逾期与归档

任务超过最晚完成时间且仍处于待办、进行中或待验收时，会被标记为逾期并触发提醒。逾期不自动取消任务，也不自动判定失败。

任务归档条件：

- 任务已完成。
- 操作者有管理该项目的权限。

项目结束条件：

- 已完成：全部阶段完成后，由项目负责人确认。
- 已取消：项目负责人或超级管理员确认非正常终止。
